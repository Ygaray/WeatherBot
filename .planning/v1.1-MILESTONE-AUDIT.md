## Tech Debt (non-blocking)

**Phase 9 — Reload Engine**
- WR-04: `/proc`-absent sentinel makes `is_weatherbot_pid` fail-open on non-Linux hosts (target host is Linux/systemd — advisory only).
- WR-02: rollback `_restore_jobs` re-invokes the same `_register_jobs` that failed; a persistent failure can leave the job table half-rebuilt (the config holder is correctly restored).
- IN-04: `test_reconcile_failure_rolls_back` patches `_register_jobs` for the whole reload, so the restore path is not genuinely exercised.

**Phase 11 — Discord Bot (integration warnings)**
- Live `[bot] operator_id` change requires restart (baked at `BotThread` construction). Single-operator personal bot — deferred for v1.
- `[reload] watch` toggle read once at startup; toggling live requires restart.
- CFG-07 "✅ config reloaded" success post does not signal that `[bot]`-section edits are restart-deferred (cosmetic).

**Note:** Phase 11's previously-flagged `ForecastCache.invalidate()` deferral is now resolved — the integration check confirms `_do_reload` invalidates the cache on committed success (`daemon.py:679-683`), so a reload no longer serves a stale forecast for up to the TTL.
