<objective>
Create the pure, surface-agnostic `weather <loc>` command parser in a NEW
`weatherbot/interactive/command.py`, with full input-matrix unit tests.

This is the parser half of Phase 6 (ROADMAP Success Criterion #3). It is config-free,
I/O-free, and validates nothing against configured locations — it only classifies raw
command text into one of three states and extracts the raw location string. The Phase 7
CLI and Phase 11 Discord bot will both feed raw text to this single function and get
identical semantics.

Purpose: One source of truth for "is this a weather command, and what location did they ask for?" — the parse-don't-validate seam (D-01) that keeps the "unknown location" signal distinct downstream for CMD-04.
Output: `weatherbot/interactive/command.py` (`parse_weather_command`, `Command`, `CommandKind`) and `tests/test_command.py`.

NOTE: This plan does NOT create `weatherbot/interactive/__init__.py`. The package barrel
is owned exclusively by plan 06-03 (so plans 01 and 02 can run in parallel without a
shared-file conflict). Until 06-03 lands, tests import directly from the submodule
(`from weatherbot.interactive.command import ...`). The directory becomes an importable
namespace package via `pythonpath=["."]` even without `__init__.py`.
</objective>
