<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator file save → observer thread → reload | The only input is a local file save by the daemon's OS user; no network/remote actor. The observer is the new component crossing this boundary. |
| observer thread → main thread | The observer (untrusted-timing) only `.set()`s a flag; the validate/swap reload runs on the main thread (no re-entrancy). |
| watched directory → secrets | `.env` may live in the same dir; the filter is the boundary that keeps secrets out of the watch surface. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Tampering / DoS | half-written/malformed file parsed mid-save | mitigate | `step=400` quiet window coalesces the save-storm into ONE yield AFTER writes settle; Phase 9 `validate_config_and_templates` rejects any partial read (keep-old). Asserted by `test_editor_save_patterns_one_reload` (SC#2). |
| T-10-02 | Information disclosure | `.env`/secret edit silently "hot-reloaded" | mitigate | `_make_watch_filter` matches ONLY config.toml + referenced templates and REJECTS `.env`; secrets are a restart boundary (Pitfall #12). Asserted by `test_env_save_never_reloads`. |
| T-10-05 | DoS (resource) | inotify-watch / fd leak over long uptime | mitigate | ONE long-lived observer started in `run_daemon`, joined in the existing `finally`; the single `watch()` generator releases fds on re-entry across a watch-set re-derive. Asserted by `test_fd_stable_and_clean_teardown` (SC#3, incl. A4 re-derive soak). |
| T-10-06 | DoS | reload loop from write-back near watched files | mitigate | Reload writes NOTHING to config/template dirs (no `.bak`, no auto-format) — Pitfall #11b. The plan adds no write-back; reload stays config-read-only. |
| T-10-07 | Tampering / DoS | re-entrant reload on the observer thread (torn state) | mitigate | Observer is flag-set-only (`request_reload` → `reload_requested.set()`); `_do_reload` runs on the main poll-loop thread only (Pitfall #6/#9). |
| T-10-08 | Elevation / Tampering | malformed save crashes the always-on daemon | mitigate | Inherited Phase 9 keep-old: the poll-loop `except` swallows a bad reload, live schedule intact. Asserted by `test_invalid_save_keeps_old_config` (SC#4). |
| T-10-09 | Tampering (low, local) | symlinked watched dir pointing outside the project | accept | Single-user local tool; `_derive_watch_dirs` resolves operator-controlled `.parent`/`TEMPLATES_DIR` only; no new external input. |
| T-10-SC | Tampering | npm/pip/cargo installs | accept | No package install in this plan (install + legitimacy gate handled in Plan 10-02). |
</threat_model>
