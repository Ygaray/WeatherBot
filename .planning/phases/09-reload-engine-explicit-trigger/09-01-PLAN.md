failure most likely to silently break a shipped guarantee. Per the AMENDED D-02 (operator Option A,
RESEARCH A3 resolved), that guard protects **name and IANA-tz edits ONLY** (both keep the slot's
send_time → same logical slot). A **send_time (or days) change is, by design, a NEW slot** — it
fires today if its new time is still ahead, then settles to the new time the next day; a separate,
explicitly-named test (`test_send_time_change_is_new_slot_fires_today_if_ahead`) pins that accepted
behavior so it can't silently regress. NO location-level same-day suppression guard is introduced
(rejected — it would break multi-slot-per-day locations). Output: a RED test suite that pins
every CFG requirement, the two-phase rollback, the diff-reconcile, the id-key migration, and the
offline check-config contract — with shared seeding/harness fixtures so the engine plans need no
test plumbing.
