<objective>
Build the reload engine in `daemon.py` (CFG-01/02/04/05/06): a SIGHUP handler that sets a
`reload_requested` Event (Pitfall 6 — flag-set only, no lock/work in the handler), a poll loop
replacing the single `stop.wait()` park so the flag is serviced on the MAIN thread (SIGTERM still
wins), and `_do_reload` — a two-phase build-then-commit: PHASE 1 calls the shared
`validate_config_and_templates` and on ANY failure logs the reason and keeps the old config+jobs
intact (CFG-04); PHASE 2 snapshots the old config + job set, `holder.replace(new_cfg)`, and
diff-reconciles APScheduler jobs on the stable `name|time|days` id (`add_job(replace_existing=True)`
/ `remove_job`, never `remove_all_jobs()`, excluding `__heartbeat__`), rolling back to the old
config + rebuilt-from-old_cfg job set on any reconcile throw (all-or-nothing, Pitfall 6/7). A
successful reload logs a `+a -r ~c =u` diff summary; a rejected one logs the reason (CFG-06/D-07).
`run_daemon` gains `config_path=` (threaded from cli.py), writes the PID file atomically on startup
(Plan 03 helper) and unlinks it in the existing `finally`. READY/.env are never touched (D-04/Pitfall 12).

Purpose: This is the phase's keystone — it turns the Plan 02 validator, the Plan 03 PID
sender/helper, and the Plan 04 stable-id key into a working, atomic, exactly-once-preserving reload.
Output: the engine + lifecycle wiring in daemon.py and the one-line `config_path` thread in cli.py,
turning ALL remaining Wave-0 reload tests green — including the load-bearing SC#4 exactly-once test
end-to-end and the all-or-nothing rollback test. Full suite green at 226 + new.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SIGHUP (OS signal) -> daemon reload path | A cross-process signal triggers a config re-read + live job mutation; it must not run re-entrantly in the handler and must not tear live state. |
| reloaded config -> live holder + APScheduler job set | A bad or mid-failing reload could leave a torn live state (half-old/half-new config or jobs). |
| reload path -> systemd READY / .env secrets | Reload must NEVER flip the health gate or re-read secrets (restart boundary). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-12 | Tampering / DoS | A malformed/partial config applied -> torn live state | mitigate | Two-phase validate-then-commit: PHASE 1 rejects on any validator raise and keeps old; PHASE 2 rolls back to old_cfg + rebuilt-from-old_cfg jobs on any reconcile throw (CFG-04, Pitfall 6). Proven by test_invalid_reload_keeps_old + test_reconcile_failure_rolls_back. |
| T-09-09 | Tampering | Exactly-once break -> duplicate/skipped briefing on a reloaded tz/name/send_time change (Pitfall #8) | mitigate | The reload never deletes sent_log rows; the Plan 04 stable-id claim makes an already-sent slot's re-fire lose. Proven by test_reconcile_diff + test_identical_reload_zero_changes. |
