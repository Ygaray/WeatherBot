<objective>
Extend `dispatch_spec` (`weatherbot/interactive/dispatch.py`) with a single,
strictly-additive keyword-only param `flags: ForecastFlags | None = None` (D-01).
When `flags is None` the existing forecast branch runs `parse_forecast_flags(arg)`
verbatim; when a caller passes a pre-built `ForecastFlags`, the parse is **skipped**
and the passed dataclass drives `lookup_name = flags.location` and
`suffix = forecast_cache_suffix(spec.name, flags)`. This is the one seam edit the
Phase-19 panel forecast path needs — Plan 02 calls `dispatch_spec(..., flags=flags)`.

Purpose: D-01 chose Option B (pass a real `ForecastFlags`, not a re-stringified arg)
so the panel is immune to location names containing flag-like tokens. The HARD
constraint (D-02) is byte-identical behavior on the `flags=None` path so the
contractual anti-drift reply suites stay green — this is a behavior-preserving
extension, NOT a refactor of the parse path.
Output: the additive `flags=` param on `dispatch_spec` + two new `test_dispatch.py`
nodes proving the passthrough and the byte-identical-when-None guarantee.
</objective>
