<objective>
Add the `CONFIG_INVALID` fatal reason (HARD-STARTUP-02, F06) as the defense-in-depth classifier: split the `run_self_check` pre-probe config/template/empty-locations checks out of the broad `except Exception` that mislabels them `NETWORK_NOT_READY`, and map the new reason to `Severity.CRITICAL`. This is the second fatal layer — the offline boot validator (29-04) is primary — but it converges on the same fatal path via the `on_fail` hook (29-05).

Purpose: Stop a config/template/empty-locations error that surfaces at probe time from warn-looping forever as a fake network fault; give the daemon an honest, CRITICAL, fatal classification.
Output: `CONFIG_INVALID` constant + classification split + severity map + re-export.
</objective>
