<objective>
Create `weatherbot/config/holder.py` with the `ConfigHolder` class — the single owner of the
live, immutable `Config` (D-04). `current()` is a lock-free atomic reference read (a bare
attribute load is one atomic bytecode under the GIL, so a reader sees the old or new WHOLE
config — never a torn one); `replace(new_config)` takes a `threading.Lock` and rebinds the
held reference. The lock serializes writers and gives Phase 9 a place to later hang an atomic
validate-then-swap. `replace()` does NOT validate in Phase 8 — that boundary is explicitly
Phase 9 (CFG-04). Canonical method name is **`replace`** (CONTEXT D-04 wins over ROADMAP's
`swap` wording — reconciled here).

Purpose: This is the seam the whole milestone's hot-reload rests on. After this plan, a future
`holder.replace(new_config)` changes what every live job reads — but only because Plan 04 wires
`fire_slot` to read `holder.current()`. This plan delivers the holder; Plan 04 consumes it.
Output: `weatherbot/config/holder.py`, turning the Plan 01 holder UNIT and CONCURRENCY tests
(`test_current_returns_held`, `test_replace_rebinds`, `test_concurrent_read_swap_safe`) GREEN.
</objective>

| T-08-05 | Tampering | Torn / partial config read under concurrency (max_workers=10) | mitigate | `current()` is a single atomic `LOAD_ATTR` under the GIL against a single atomic `STORE_ATTR` in `replace()`; a reader always observes a whole, previously-validated, frozen Config — never a torn one.
