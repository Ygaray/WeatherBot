<objective>
Implement the seven read-only command handlers and the read-only `DaemonState` accessor that `status` needs. The weather views (`alerts`/`sun`/`wind`/`next-cloudy`) read fields off the already-fetched One Call payload retained on `Forecast`; the info handlers (`help`/`locations`) read the registry/config; `status` reads live scheduler + heartbeat + liveness through a small injected read-only accessor.

Purpose: This is the body of the command surface — CMD-10/11/12/13/14/15. Every handler is store-free (D-06 / SC#5) and surface-agnostic (returns content both the Discord embed and CLI plain-text rendering consume, D-04). The handlers are wired into the bot/CLI dispatch and threaded through the daemon in Plan 03.

Output: a new `interactive/commands/` package, a `DaemonState` accessor in `interactive/state.py`, and their tests.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| One Call payload → handlers | Remote JSON read off the retained `Forecast` payload; handlers must use defensive `.get()`/`or []` |
| DaemonState → status | Live scheduler/db/start-time injected read-only into the command layer; must never be a write path |
| Handlers → SQLite store | HARD boundary: handlers must NOT cross it (read-only, D-06 / SC#5) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-06 | Tampering | a handler writing the store | mitigate | No store import in any handler; zero-store-writes spy extended to cover all new handlers (D-06 / SC#5) |
| T-12-07 | Elevation of Privilege | status mutating config/scheduler | mitigate | DaemonState is read-only — no add_job/remove_job/holder.replace/stamp_*; asserted by grep gate (D-02 "reports, never mutates") |
| T-12-08 | Information Disclosure | secrets in status/alerts reply | mitigate | status reports state only (never config secrets/token/appid); alerts surfaces event text only — no URL/key |
| T-12-09 | DoS | a handler raising on a malformed payload | mitigate | Defensive `.get()`/`or []` (Pitfall 2 pattern); the bot envelope (Plan 03) catches any escape (CMD-16) |
| T-12-SC | Tampering | npm/pip/cargo installs | mitigate | None — no new packages added |
</threat_model>
