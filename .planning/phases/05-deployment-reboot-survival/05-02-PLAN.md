<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| systemd → daemon process | The OS supervisor starts/stops/restarts the process and injects `$NOTIFY_SOCKET` + `EnvironmentFile=` env |
| daemon → OpenWeather API | Untrusted network response crosses into the startup self-check probe |
| daemon → Discord webhook | Outbound one-time "online" ping text |
| host filesystem → secrets | `.env` carries the OpenWeather key + Discord webhook URL |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-ID-03 | Information Disclosure | secrets in unit file / journald | mitigate | Secrets ONLY via `EnvironmentFile=.env` (never inline `Environment=KEY=...`); `chmod 600 .env`; outcome-only daemon logging; health row carries reason/detail only (Task 2/3 acceptance). |
| T-05-DoS-02 | Denial of Service (self-inflicted) | `Type=notify` start-timeout + Restart=always | mitigate | `TimeoutStartSec=infinity` so the deferred-online gate cannot become a disguised crash-loop (Pitfall 1, Task 2). |
| T-05-DoS-03 | Denial of Service (self-inflicted) | `network-online.target` misread as "internet up" | mitigate | The re-probe loop classifies first-boot DNS/connection errors as `network_not_ready` and waits rather than treating them as fatal (Pitfall 4, Task 1, D-06). |
| T-05-DoS-04 | Denial of Service (self-inflicted) | crash-loop on failed self-check | mitigate | Daemon NEVER exits/raises on a failed self-check — it stays alive and re-probes (D-04, Task 1). |
| T-05-DoS-05 | Denial of Service (self-inflicted) | `WatchdogSec` without `WATCHDOG=1` | mitigate | No `WatchdogSec` set in v1 (Pitfall 6, Task 2 acceptance asserts its absence). |
| T-05-DoS-06 | Denial of Service (self-inflicted) | `systemctl stop` ignored during re-probe loop → SIGKILL after TimeoutStopSec | mitigate | SIGTERM handler registered BEFORE the gate; loop blocks on `stop.wait(interval)` so stop breaks it immediately (Pitfall 2, Task 1). |
| T-05-T-02 | Tampering | Discord "online" ping markdown/mention injection | mitigate | The ping is a FIXED literal string with no user/template interpolation and no `@everyone`/mention (Task 1). |
| T-05-EoP | Elevation of Privilege | running the daemon as root | mitigate | `User=<non-root>` in the unit (V4 least-privilege, Task 2 acceptance). |
| T-05-SC | Tampering | npm/pip/cargo installs | mitigate | This plan installs ZERO new packages (systemd is a host facility; sd_notify is stdlib from Plan 05-01). No legitimacy checkpoint needed. |
</threat_model>
