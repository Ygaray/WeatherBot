<objective>
Build the three Phase-5 foundation pieces the daemon gate (Plan 05-02) will wire
together, with no daemon changes yet: (1) a classified self-check engine extracted
from `do_check`, (2) a pure-stdlib `sd_notify` READY=1 helper, and (3) a single-row
`health` table + `stamp_health` upsert in the SQLite store. Refactor `do_check` to
call the shared engine so `--check` and the daemon use ONE implementation (D-03).

Purpose: OPS-02's startup self-check + online signal need a reusable, *classified*
validate+probe (pass / network_not_ready / auth_failed), a way to tell systemd it
is ready, and a durable queryable health row — all independently testable before any
daemon wiring. Splitting these into Wave 1 keeps the daemon-wiring plan (05-02) small
and lets these land with their own Wave-0 tests.

Output: `weatherbot/ops/` package (`__init__.py`, `sdnotify.py`, `selfcheck.py`),
an additive `health` table + `stamp_health` in `store.py`, a `do_check` refactor in
`cli.py`, and three test files (`test_ops_selfcheck.py`, `test_sdnotify.py`, extended
`test_store.py`).
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| daemon/cli → OpenWeather API | Untrusted network response (status code, body, Retry-After) crosses into the self-check probe |
| app → SQLite (`health` row) | App-written values persisted; must never carry secrets |
| app → systemd (`$NOTIFY_SOCKET`) | App writes a readiness datagram to an OS-provided socket path |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-ID-01 | Information Disclosure | `health` row `detail` column, self-check logs | mitigate | `detail` is outcome-only (status code / exception class name); parameterized SQL only; reuse outcome-only logging (T-04-01). No-secret test asserts no key/webhook substring in any health column (Task 1). |
| T-05-ID-02 | Information Disclosure | self-check 401/403 message | mitigate | Reuse existing do_check wording that names the failure class without echoing the key/URL/params (Task 3). |
| T-05-T-01 | Tampering | `health.reason`/`detail` write path | mitigate | Parameterized `UPDATE ... WHERE id=1` only; never f-string a value into SQL (Task 1). |
