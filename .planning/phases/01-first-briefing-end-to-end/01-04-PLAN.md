| process → Discord webhook | The webhook URL is a bearer credential; posting + logging must not leak it |
| `.env`/env → composition root | API key + webhook URL injected via Settings; never logged or echoed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Information Disclosure | Webhook URL / API key in structlog or traceback | mitigate | send path logs outcome only; never logs the webhook URL or `appid`; redact in structlog — asserted in tests |
| T-04-02 | Spoofing | Leaked webhook URL lets anyone post as WeatherBot | accept | Treated as a credential (gitignored `.env`); rotation is the user's remedy; out of Phase 1's active surface but documented (RESEARCH Security Domain) |
