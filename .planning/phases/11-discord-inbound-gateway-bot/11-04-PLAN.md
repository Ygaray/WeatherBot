## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| bot thread → daemon process | a bot/gateway failure must not cross into the scheduler/briefing path |
| bot thread → systemd READY gate | bot health must never flip the readiness signal |
| reload path → Discord channel | a reload-outcome post must not abort the reload |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-11 | Denial of Service (of core value) | bot/gateway failure stops scheduled briefings | mitigate | Bot on its own daemon thread started AFTER emit_online; start wrapped in try/except that proceeds; BotThread._run catches LoginFailure + Exception and dies alone (11-03); scheduler thread + briefing path untouched. CMD-08 SC#4: revoke token → next briefing still fires |
| T-11-12 | Tampering | bot health flips the systemd READY gate | mitigate | emit_online/notifier.ready() fires ONCE in run_daemon before the bot starts and is never called from the bot path (acceptance-asserted); a dead bot thread does NOT touch gate_until_healthy (Pitfall 4) |
| T-11-13 | Denial of Service | a failing reload-outcome post aborts the reload | mitigate | CFG-07 posts wrapped in best-effort try/except (emit_online idiom) — a channel.send failure is logged, never propagated; reload success/keep-old is independent of the post (best-effort test) |
| T-11-14 | Tampering | off-main-thread signal handler crashes the bot thread (set_wakeup_fd ValueError) | mitigate | Bot uses asyncio.run(client.start()) NOT client.run(); daemon keeps its main-thread signal.signal handlers; no add_signal_handler added (acceptance-asserted) |
</threat_model>
