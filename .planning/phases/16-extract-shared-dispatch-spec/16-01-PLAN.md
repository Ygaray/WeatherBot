<objective>
Lift the heterogeneous arg-adaptation if/elif ladder out of BOTH `on_message`
(`bot.py:313-329`) and its identical twin in `cli.py:_run_registry_command`
(`cli.py:618-632`) into ONE shared dispatcher (`weatherbot/interactive/dispatch.py`),
so the registry command set can never drift across surfaces (PANEL-10). The async
surface (bot, and the Phase-17 panel) routes through `dispatch_spec` (off-loop fetch
→ ladder); the sync CLI routes through `dispatch_reply` (the bare ladder). This is a
PURE, behavior-preserving refactor: replies stay byte-identical, proven by the
existing contractual test suite staying green.

Purpose: Drift-prevention groundwork (criterion #3) laid BEFORE any panel callback
exists (Phase 17), so the panel can only ever be a third caller of the same code.

Output: New module `weatherbot/interactive/dispatch.py` with `dispatch_reply` (sync)
and `dispatch_spec` (async); `bot.py` and `cli.py` re-routed through it; a small
test asserting both surfaces resolve via the shared dispatcher; the full
`uv run pytest` suite green and byte-identical.
</objective>
