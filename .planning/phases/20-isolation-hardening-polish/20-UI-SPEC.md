# Phase 20 — UI Design Contract

> Visual and interaction contract for the Discord operator panel polish (PANEL-12 / PANEL-13).
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Surface note (read first).** This is **not** a web/HTML UI. The surface is a
> discord.py 2.7.1 operator panel: message **embeds** + interaction **components**
> (Buttons, one Select dropdown). There is no CSS, no font stack, no color palette
> beyond Discord's single embed accent bar. The standard UI-SPEC dimensions are
> **re-mapped** to Discord-native concepts:
>
> | UI-SPEC dimension | Discord-native mapping for this phase |
> |---|---|
> | Spacing | Embed **line ordering** in the description block; component **row/grid** layout (5×5 hard cap) |
> | Typography | Discord's **native** embed rendering (title / description / field name·value / footer). Not author-controlled — declared as roles, not px/weights |
> | Color | The **single** embed accent bar (`BRIEFING_COLOR_INT`) + **emoji as the only per-control color signal**. Meaning is carried by text + emoji, **never** button `style` color |
> | Copywriting | Embed copy: the `📍` indicator line, the `Updated <t:…>` stamp, emoji-coded button labels |
> | Registry | discord.py component API only. **No third-party block registry** (not a shadcn project) |
>
> **All visual decisions are LOCKED** in `20-CONTEXT.md` (D-01..D-07). This contract
> **formalizes** them into checkable assertions; it does **not** reopen them.

## Spacing Scale

> Re-mapped: there is no px spacing. "Spacing" = (a) embed **line order** in the description
> block and (b) the component **row grid**. Both have hard, checkable rules.

### Embed description line order (the result embed body)

The polish adds two lines to the embed **description** (never the title — `<t:>` markdown does
not render in a title, D-07). Declared order, top of description:

| Order | Line | Condition | Source |
|-------|------|-----------|--------|
| 1 | `📍 {selected_location}` | Location-bearing replies only (weather/uv/next-cloudy/sun/wind/forecast). **Suppressed** on argless replies (status/alerts) — pass `location=None` | D-01 |
| 2 | `Updated <t:{unix}:t> (<t:{unix}:R>)` | **Every** reply (location-bearing and argless) | D-06 |
| 3+ | existing reply fields (`add_field` loop) | unchanged | existing |

- `unix = int(discord.utils.utcnow().timestamp())`, computed **per render**.
- When `📍` is suppressed (argless), the description is the `Updated …` line alone — no blank leading line.
- Whether `📍` and `Updated` share one description block or sit on separate lines is at executor discretion **provided line order above holds** and both render in description/field/footer-text (never title). Recommended: two lines joined by `\n` in one description block.

### Component row grid (the panel itself — unchanged this phase)

| Rule | Value | Status this phase |
|------|-------|-------------------|
| Max rows | 5 | unchanged — no new row added |
| Max components per row | 5 | unchanged |
| Max children total | 25 | unchanged — grid is **5/5 full** when forecast sub-grid is revealed |
| `custom_id` length | ≤ 100 | unchanged |
| label length | ≤ 80 | unchanged |

**Exceptions / hard constraint:** This phase adds **ZERO new component slots**. The `📍`
indicator, the `Updated` stamp, and all emoji are **non-component** additions (embed text +
a per-button `emoji=` param). `_assert_layout` (`panel.py`) must remain green and untouched —
adding any slot would trip the 5/5 guard. Verify nothing new trips it.

## Typography

> Re-mapped: Discord renders these roles natively; the author does not set px/weight/line-height.
> Declared as **role → embed region** so the checker can verify the right content lands in the
> right region (the only author-controllable typographic decision here).

| Role | Embed region | Author-controlled? | Rule |
|------|--------------|--------------------|------|
| Heading | Embed **title** | content only | Reply title (e.g. "Weather — {loc}"). **No `<t:>` token** here (does not render, D-07). No emoji baked here. |
| Body | Embed **description** | content + line order | Carries the `📍` line + `Updated <t:…>` line per the Spacing line-order table. |
| Label | Embed **field name** | content only | Existing reply field names — unchanged. |
| Value | Embed **field value** | content only | Existing reply field values — unchanged. |
| Footer / meta | Embed **footer + native timestamp** | retained | Keep `embed.timestamp = discord.utils.utcnow()` (`bot.py:260`, D-07) — a second always-correct time signal; zero cost, redundant-but-harmless. |
| Control label | Button **label** + **emoji** | full | Title-Case text label (kept) **plus** a leading `emoji=` glyph. See Copywriting Contract. |

**Control-label rule (D-04):** emoji is applied via the **separate** discord.py `emoji=`
param — **never** concatenated into the label string. The client renders icon + text with
correct native spacing and the text label is retained for screen-reader naming and to
disambiguate confusable buttons.

## Color

> Re-mapped: there is exactly **one** color (the embed accent bar). Per-control differentiation
> is carried by **emoji + text only**, never by button `style` color. This is the load-bearing
> color rule for the panel.

| Role | Value | Usage |
|------|-------|-------|
| Embed accent bar | `BRIEFING_COLOR_INT` (existing, `bot.py`) | The single left-edge color stripe on every result embed. **Unchanged this phase.** |
| Button style | `discord.ButtonStyle.primary` (uniform) | All command + forecast + toggle buttons are uniform `primary`. Meaning is carried by **label + emoji**, never per-button color (confirmed in `ForecastButton` docstring, `panel.py:196-197`). **No `destructive`/red styles** — the panel is read-only; there are no destructive actions. |
| Emoji as signal | the D-05 glyph set | The **only** per-control visual differentiator. Deliberate semantic pairs: ☁️/☀️ (cloud↔sun), 🟢-status (steady-state health) vs ⚠️-alerts (exception signal). |

**Accent reserved for:** the single embed accent bar (`BRIEFING_COLOR_INT`) only. There is
**no** secondary/destructive color in this surface. Button color carries **no** meaning and
MUST stay uniform `primary`.

## Copywriting Contract

> This is the substantive contract for the phase. Every string below is locked or
> executor-discretion-within-bounds; the checker validates exact glyphs and token shape.

### Emoji-coded button labels (PANEL-13a — D-04/D-05, LOCKED "keep all")

Each button keeps its Title-Case text label (`_LABELS`, `panel.py:105-113`) **and** gains a
leading `emoji=` glyph. The locked set (single-codepoint / well-supported glyphs, user
reviewed flagged picks and chose **keep all**):

| Control | Text label (kept) | Emoji (`emoji=`) |
|---------|-------------------|------------------|
| weather | Weather | 🌡️ |
| uv | UV | 🧴 |
| next-cloudy | Next Cloudy | ☁️ |
| sun | Sun | ☀️ |
| wind | Wind | 💨 |
| status | Status | 🟢 |
| alerts | Alerts | ⚠️ |
| Forecast (toggle) | Forecast | 📅 |
| Weekday Detailed | Weekday Detailed | 📋 |
| Weekday Compact | Weekday Compact | 📝 |
| Weekend Detailed | Weekend Detailed | 🏖️ |
| Weekend Compact | Weekend Compact | 🌴 |

- Recommended implementation: a parallel `_EMOJI` dict keyed like `_LABELS`, plus per-construction emoji on the forecast/toggle buttons (executor discretion: parallel dict vs inline — D-04/D-05 discretion).
- **Clone-survival contract (THE trap — Pitfall 1):** `_render_view` (`panel.py:654-701`) rebuilds bare Button/Select clones that today copy only `label/custom_id/style/row/disabled`. The clone path MUST also carry `emoji=child.emoji` on every Button clone — otherwise emoji **vanish** on every disabled-ack and collapse render. A test MUST assert emoji survive the clone (not just the freshly-built `__init__` view).

### Selected-location indicator (PANEL-12 — D-01/D-02)

| Element | Copy / behavior |
|---------|-----------------|
| Indicator of record (embed line) | `📍 {selected_location}` rendered on **every** location-bearing render, derived from `self._selected_location` at render time (restart-safe by construction, zero component slots). **This is the guarantee.** |
| Dropdown highlight (decorative reinforcement) | The current location's `SelectOption(default=True)`, re-applied from `_selected_location` on **every** view rebuild (each edit AND restart / `add_view` reconstruction). **Best-effort only**, NOT source of truth. |
| Startup default | `locations[0]` (home/first) — already implemented (`panel.py:319`); PANEL-12 only makes it **visible**, no change to default-resolution logic (D-03). |

- **Never** read selection back from `Select.values` (empty for default options until actively changed — discord.py #7284). Selection lives **only** in `_selected_location` (D-02).
- **Clone-survival contract:** the `default=True` re-mark MUST be rebuilt in the `_render_view` Select clone (`panel.py:691-700`) — rebuild options as `SelectOption(label=n, value=n, default=(n == self._selected_location))`. Forgetting it silently reverts the dropdown to bare `placeholder`. A test MUST assert the default mark survives the clone.

### "Updated" stamp (PANEL-13b — D-06/D-07)

| Element | Copy |
|---------|------|
| Stamp line (description) | `Updated <t:{unix}:t> (<t:{unix}:R>)` where `unix = int(discord.utils.utcnow().timestamp())` per render |
| Rendering | `:t` = operator's local clock time in their own 12h/24h preference (no server tz/DST math); `:R` = relative clause Discord re-renders ~every minute, snapping back to "now" on each in-place edit (satisfies "visibly distinct from the prior render"). |
| Placement | Embed **description** / field / footer-text — **NEVER** the title (`<t:>` does not render there, D-07). |
| Native timestamp | Keep `embed.timestamp = discord.utils.utcnow()` (D-07) — retained, redundant-but-harmless second signal. |

### Standard copywriting slots (mapped to this surface)

| Element | Copy |
|---------|------|
| Primary CTA | Per-command emoji-coded buttons (e.g. "🌡️ Weather", "📅 Forecast"). No single CTA — the panel is a button grid of equal-weight read-only triggers. |
| Empty state | **N/A** — the panel always has a valid selected location (D-03 default makes a no-location state unreachable); no empty data state exists for this polish. |
| Error state | Existing identity-free best-effort reply `"Sorry — something went wrong."` (`_ERROR_REPLY`, `panel.py:118`) via the per-callback envelope — **unchanged this phase**. The `📍`/`Updated` lines must not leak into or alter the error path's leak-free contract. |
| Destructive confirmation | **N/A** — the panel is strictly read-only; there are no destructive actions in this surface (or the milestone). |

## Interaction & Isolation Contract (surface-specific)

> Not in the web template, but load-bearing for this Discord surface. The checker should treat
> these as part of the visual/interaction contract.

| Contract | Rule | Source |
|----------|------|--------|
| In-place render | Every result edits the **same** panel message (`edit_original_response`); no new messages posted. The `📍`/`Updated` lines render inside this existing path. | Phase 17/19 |
| Single-ack discipline | One `response.edit_message` cue (`⏳ Fetching…`) then one `edit_original_response` result — never a second `response.*`. The polish adds **no** new ack. | D-14 (prior) |
| Render on EVERY path | The `📍` line, `Updated` stamp, emoji, and dropdown re-mark must apply on **every** render path: ack, result, collapse, and error edit. | D-01/D-02/D-06 |
| Failure isolation (PANEL-11) | A panel callback that **raises or hangs** never delays, drops, or stops a concurrently-scheduled briefing. **No production change** to the isolation path — proven by a live-`BackgroundScheduler` test (hang via `await asyncio.Event().wait()`, D-08a) + a D-08b executor-sharing audit. | D-08/D-09 |
| No callback timeout | `asyncio.wait_for` around callbacks is **deliberately NOT added** (D-09) — out of scope, regression-averse. | D-09 |
