<objective>
The config + templating slice: give `Location` an authoritative IANA `timezone` and an optional per-location `units` override with fail-loud validators (D-03, LOC-02, CONF-03), ship a `config.example.toml` with ‚Č•2 real locations (LOC-01, CONF-01), and add the template-placeholder validation layer (`validate_template`) that wraps ‚ÄĒ never replaces ‚ÄĒ the guarded renderer (D-10) and aborts the send on a typo'd placeholder at every load including `--send-now` (D-11, TMPL-02). After this slice the user can configure two independent locations and a typo in the template fails loudly instead of shipping blank.

Purpose: This makes the configuration "real" (multi-location, per-location tz/units) and closes the templating-safety gap. It depends on Plan 02 because `from_payloads`/`placeholders()` now read `loc.timezone` and expose the new placeholders that `CANONICAL` must match exactly.
Output: Extended `Location` model + validators, a unique-name helper, `validate_template` + `CANONICAL`, updated starter templates + `config.example.toml`, and validation wired at the `send_now` load boundary.
</objective>
