<objective>
The core vertical slice: migrate the data source fully from the two free 2.5 endpoints + 3-hour bucket aggregation to a single One Call 3.0 fetch (D-01), and use the richer payload to deliver the new briefing content — `{feels_like}` (D-05), five threshold-driven `{hint}`s (D-06/07), and a passive severe-weather `{alert}` line (D-08). After this slice, `--send-now` again produces a complete, correct briefing — now One-Call-sourced and feature-rich — and every fetch persists to a new `weather_onecall` table.

Purpose: This is the headline rework of the phase and the foundation every other slice builds on. It repoints the API client, rewrites the `Forecast` mapping (Plan 01 already severed the dead `aggregate` import to keep collection green; this plan supplies the REAL One Call `daily[0]` high/low/rain plus feels_like/hint/alert), reshapes persistence, and collapses the send path from 4 calls to 2 (the dual-unit strategy is TWO One Call fetches — imperial + metric — per D-02, chosen over one-call-and-convert to avoid conversion drift) — all while preserving the Phase 1 DATA-03 single-fetch-reuse and secret-safe-logging discipline.
Output: A One Call 3.0 client (+ geocode helper), a rewritten `Forecast.from_payloads` with hint/alert derivation, a `weather_onecall` store table, and a 2-call `send_now`.
</objective>
