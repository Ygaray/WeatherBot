## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user CLI input (`--geocode "query"`) → Geocoding API | User-supplied text passed only as an httpx query param (no shell/SQL). |
| `--check` → OpenWeather One Call API | A single live reachability call carrying the secret `appid`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Information disclosure | cli.py (--check/--geocode) | mitigate | Inherit the secret-safe discipline: httpx logger pinned WARNING; `--check`/`--geocode` error messages never echo the key/URL/params (Pitfall 6). |
| T-04-02 | Spoofing / Info-disclosure | --check reachability probe | mitigate | Distinguish a 401/403 "subscription not active or not yet propagated" message from a generic error WITHOUT leaking the key (Pitfall 1); a single best-effort call, not retried (retry is Phase 4). |
| T-04-03 | Tampering | --geocode handler | mitigate | `--geocode` prints only and NEVER writes the config file (D-04); the send path never geocodes (LOC-03), so quota protection holds by construction. |
| T-04-04 | Input validation | --geocode query | accept | The query is passed only as an httpx `params` value — no shell, no SQL, no eval; no injection surface. |
| T-04-SC | Tampering | package installs | accept | No new packages (RESEARCH "Package Legitimacy Audit": none); no install/slopcheck gate required. |
