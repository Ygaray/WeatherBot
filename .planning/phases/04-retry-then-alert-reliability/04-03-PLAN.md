<objective>
Wire the daemon's patient reliability path end-to-end: wrap `fire_slot`'s delivery in the Plan-01 two-burst retry (threading the daemon's existing `stop_event` for an interruptible pause), classify the outcome into the reason taxonomy and write the deduped `briefing_missed` alert + CRITICAL log, resolve the alert + stamp heartbeat success on eventual delivery, harden the existing exception isolation to also alert (`internal_error`) with a full traceback while the scheduler survives, and register a periodic heartbeat tick in `run_daemon`.

This is the vertical slice that makes the whole phase observable: after this plan a real always-on daemon recovers from transient blips, surfaces genuine failures out-of-band, and proves liveness. Depends on Plan 01 (retry engine + `REASON_*`) and Plan 02 (store helpers).

Purpose: Implement the daemon (unattended) half of RELY-01..06 per D-01 (out-of-band alert = CRITICAL log event + durable DB row, never Discord), D-07 (two-burst), D-08 (auth short-circuit + classification), D-10 (daemon-only patient path), D-11 (dedup anti-loop), D-12 (exception→alert + traceback), D-13 (resolve on success), D-04/D-05/D-06 (heartbeat tick + success stamp + log event).
Output: modified `weatherbot/scheduler/daemon.py`; the corresponding behavior tests un-skipped/filled in `tests/test_reliability.py`.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| OpenWeather/Discord failures → fire_slot | Untrusted I/O outcomes drive retry/alert classification |
| fire_slot → alerts/heartbeat rows + logs | Outcome data crosses into durable records a future monitor reads |
| one bad slot → scheduler thread | An exception must not cross the per-job boundary and kill the loop |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Information Disclosure | new `briefing_missed`/`heartbeat` log fields | mitigate | Flat outcome-only fields (location/slot/local_date/reason/severity); test greps captured logs for `appid`/host, asserts absent (Task 1/2) |
| T-03-07 | Denial of Service | one bad job killing the scheduler | mitigate | Hardened `except Exception` returns None + alerts; thread survives (RELY-06 / D-12) |
| T-04-DoS | Denial of Service | unbounded retry burning quota / blocked shutdown | mitigate | Config-driven bounded two-burst budget (~65 min < 90-min grace); `sleep=stop_event.wait` makes the pause interruptible (D-07 / Pitfall 1) |
| T-04-LOOP | Denial of Service | alert loop on restart-within-grace | mitigate | `record_alert` INSERT-OR-IGNORE dedup (D-11); alert path never touches Discord (D-02) |
| T-04-LOG | Tampering | log-injection via reason/location fields | mitigate | Reason strings are fixed constants from Plan 01; location is config-validated; structlog renders flat kwargs (no format string) |
</threat_model>
