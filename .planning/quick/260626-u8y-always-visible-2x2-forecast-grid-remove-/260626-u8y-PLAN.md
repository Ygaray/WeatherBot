<objective>
Replace the v1.3 panel's two-tier Forecast interaction (a `📅 Forecast` toggle button that
reveals a hidden 2×2 sub-grid) with an **always-visible 2×2 forecast grid**. This is an
approved UX change requested by the product owner during live Gate-2 UAT: the extra expand
click is removed and the four forecast variant buttons are permanently shown. The panel
becomes a fixed 12-child / 5-row layout, exactly filling Discord's 5-row max:

```
row0: [ Location: <selected> ▼ ]                          (LocationSelect)
row1: 🌡️Weather 🧴UV ☁️Next-Cloudy ☀️Sun 💨Wind            (5 CmdButtons)
row2: 🟢Status  ⚠️Alerts                                   (2 CmdButtons — Forecast TOGGLE REMOVED)
row3: 📋Weekday Detailed   📝Weekday Compact               (2 ForecastButtons)
row4: 🏖️Weekend Detailed  🌴Weekend Compact               (2 ForecastButtons)
```

Purpose: ship the approved Gate-2 UX change (drop the expand click) while PRESERVING the
just-shipped clone-routing fix (commit b48abc6 / debug `panel-dead-after-first-tap`) — the
single hardest invariant here. The `_render_view` clone bound to the panel message MUST keep
carrying LIVE callback-bearing item subclasses, or every tap after the first goes dead again.

Output: a simplified `panel.py` (no toggle, no `_expanded` state machine, no `expanded=` knob),
updated `tests/test_panel.py` (obsolete toggle/collapse tests removed; always-visible grid +
cloned-forecast-routing assertions added; clone-routing regressions kept), and annotated
PANEL-07 / Phase-19 planning docs.
</objective>
