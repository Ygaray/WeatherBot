<objective>
Build the Phase 12 contract layer: the single self-describing command registry, the generalized registry-driven parser, and the three data seams the read-only command handlers depend on (widened One Call `exclude` to keep `hourly`, read-only heartbeat/health store readers, and a declared cloud-cover threshold config knob).

Purpose: Every downstream handler and both surfaces (CLI + Discord) derive from ONE registry list so `help` can never drift (CMD-09), and `next-cloudy` (CMD-15) plus future Phases 14/15 require the `hourly[]` data that v1.0 trimmed. These are the interface contracts the later plans build against — defining them first prevents a scavenger hunt.

Output: `registry.py` (new), a generalized `parse_command` in `command.py`, widened `client.py`, new store readers, a config knob, and their tests.
</objective>
