<objective>
Lay the durable foundation for the UV monitor: extend the existing `[uv]` config table with the three monitor-only knobs, add a dedicated `uv_alerts` dedup table + claim/read helpers to the store, promote `_fires_on` to a public `fires_on`, and stand up the `tests/test_uv_monitor.py` Wave-0 scaffold with a build-time dependency canary.

This plan touches NO monitor logic and NO daemon wiring — it only creates the primitives Plan 15-02 (the tick) and Plan 15-03 (the daemon registration) consume. Keeping it isolated means the dedup namespace (UV-05) and the config surface land before any code that fires alerts.

Purpose: Make the once/day/location dedup restart-durable (UV-05, Pitfall 2), expose the live-reloadable monitor config (UV-04), and reuse the single source-of-truth active-today logic (no forked weekday code).
Output: extended `UvConfig`, `uv_alerts` table + `claim_uv_alert`/`claimed_uv_kinds`, public `fires_on`, and the Wave-0 test file + canary.
</objective>
