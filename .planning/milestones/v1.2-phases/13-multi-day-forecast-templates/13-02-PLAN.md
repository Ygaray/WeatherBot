| Boundary | Description |

| user-editable template file → rendered output | A typo'd or hostile `{token}` could ship a literal placeholder or attempt injection |
| forecast data → Discord/CLI text | Rendered forecast eventually posts to a channel |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-04 | Tampering / EoP | template injection via editable `.txt` | mitigate | `render_forecast` routes BOTH header and per-day line through the existing guarded `render`/`validate_template`; no `str.format`/`Formatter`/`eval`; only `{name}` whitelist substitution (renderer invariant) |
| T-13-05 | Tampering | typo'd `{token}` shipping a literal placeholder | mitigate | `validate_template(text, allowed=FORECAST_TOKENS)` and `validate_template(line_fmt, allowed=day_allowed)` fail loud at load before any render (Pitfall 5) |
| T-13-06 | Information Disclosure | secret leaking into a forecast template | accept | Forecast token sets contain only weather/label tokens; no appid/webhook/token token exists, so a template cannot reference a secret |
</threat_model>
