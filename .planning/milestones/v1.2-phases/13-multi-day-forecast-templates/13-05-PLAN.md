and fully isolated from the briefing. Depends on Plans 01/02/03 (Wave 2; parallel with Plan 04 —

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| config.toml forecast slots → scheduler | User-editable slot values become live cron jobs |
| user-editable forecast template → scheduled send | A typo'd template could crash a scheduled fire |
| scheduled forecast fire → briefing spine | A forecast failure must not cross into / gate the briefing path |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-13-15 | DoS | a forecast slot fire crashing the scheduler thread / gating a briefing | mitigate | `fire_forecast_slot` body wrapped in `fire_slot`'s try/except (log + return `None`); forecasts run as independent jobs — a forecast failure never touches the briefing's `claim_slot` path (UV-06-style isolation, FCAST-05) |
| T-13-16 | Tampering | a scheduled forecast writing the SQLite time series | mitigate | `fire_forecast_slot` calls no `claim_slot`/`release_claim`/store function and imports nothing from the store (A1 no-claim/no-catchup); proven by the zero-store-writes spy (FCAST-05) |
| T-13-17 | Tampering / DoS | a malformed forecast template crashing a scheduled fire | mitigate | Forecast templates validated by `validate_config_and_templates` at load + reload keep-old; a bad template is rejected before it can swap (Pitfall 5) |
