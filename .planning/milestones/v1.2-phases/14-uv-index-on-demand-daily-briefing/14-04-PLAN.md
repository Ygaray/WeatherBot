<objective>
Add the on-demand `uv <loc>` command (UV-01) to both surfaces. A new read-only
`uv` handler reads the already-fetched One Call payload off the shared
`LookupResult`, calls the Plan 14-02 `compute_uv`, and returns a surface-agnostic
`CommandReply` carrying the full summary set PLUS a compact daytime hourly UV line
(D-04 — richer than the briefing). It registers as one `CommandSpec` on the Phase-12
registry (so CLI subparser + Discord dispatch + `help` pick it up automatically) and
both dispatch sites thread `config.uv.threshold` into it, mirroring `next-cloudy`.

Purpose: closes UV-01 — current + max UV on demand from both the CLI and the Discord
bot — reusing the entire Phase-12 command machinery (guard ladder, read-only lookup
core, off-loop fetch, embed/text render) with zero new infrastructure.

Output: `uv` handler in `weather_views.py`; `uv` registry spec + wiring; the
`config.uv.threshold` dispatch branch in `cli.py` and `bot.py`; tests across the
registry, handler, CLI, and bot.
</objective>
