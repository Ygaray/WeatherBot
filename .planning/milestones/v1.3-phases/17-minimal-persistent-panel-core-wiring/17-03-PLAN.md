<objective>
Build `weatherbot/interactive/panel.py`: a single `PanelView(discord.ui.View,
timeout=None)` with static `custom_id`s — a location `Select` (row 0), the five
location-command buttons (row 1: weather · uv · next-cloudy · sun · wind), and the
two argless buttons (row 2: status · alerts) — each derived from a curated tuple
through `registry.BY_NAME`. The panel is the THIRD caller of the Phase-16
`dispatch_spec` seam. This file carries the load-bearing interaction correctness:
the single-ack defer-then-edit contract (D-14/D-15), the `interaction_check`
operator gate with explicit reject log (D-11/D-12/D-13), and the per-callback
non-propagating envelope plus `View.on_error` backstop. All eleven `test_panel.py`
node IDs (Plan 01) go GREEN here.

Purpose: This is the whole point of Phase 17 — wire a tap-to-drive panel onto the
existing read-only command core without adding a new dep, intent, client, or
on-loop I/O, and without ever letting a callback failure reach the briefing
scheduler. The three correctness mechanisms are non-negotiable and verified against
discord.py 2.7.1 source (17-RESEARCH Patterns 1–4).
Output: `weatherbot/interactive/panel.py`.
</objective>

| Boundary | Description |
|----------|-------------|
| Discord gateway → PanelView callback | Component interaction (INTERACTION_CREATE) crosses here; the tapper may be a non-operator |

| callback exception → gateway loop / scheduler thread | A raised callback must not propagate past the panel (briefing-isolation seam) |

| T-17-03-03 | Denial of Service | double-tap / re-tap during a cold fetch | mitigate | The single `response.edit_message` ack disables all components (`_disabled_copy`) before the fetch, neutralizing re-taps; exactly one `response.*` per tap prevents `InteractionResponded`. |
| T-17-03-04 | Denial of Service | callback exception leaking onto the briefing scheduler thread | mitigate | Per-callback non-propagating `try/except` + `View.on_error` backstop; the panel touches only read-only registry + ForecastCache + holder.current(); never the scheduler or holder.replace. | T-17-03-06 | Denial of Service | blocking the gateway loop | mitigate | No on-loop httpx/sleep; all blocking work stays off-loop inside `dispatch_spec`'s `run_in_executor`. |
| T-17-03-SC | Tampering | npm/pip/cargo installs | accept | No package install (RESEARCH §Package Legitimacy Audit: N/A); discord.py already pinned, no new dep/intent. |
