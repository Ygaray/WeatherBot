<objective>
Land the two render-layer polish additions in the SINGLE surface-agnostic embed builder
`render_embed` (`bot.py`): a `📍 {selected_location}` indicator line (PANEL-12, D-01) and
an `Updated <t:{unix}:t> (<t:{unix}:R>)` self-ageing stamp (PANEL-13b, D-06), both in the
embed **description** (never the title — `<t:>` does not render there, D-07). The selected
location is threaded in via a default-`None` keyword arg so every existing caller stays
source-compatible; the `📍` line is suppressed when `location is None` (argless replies).

Purpose: Both additions live in the one shared render path so panel / bot / CLI cannot
drift (research Architectural Responsibility Map). The `<t:>` token off-loads tz/DST/12h-24h
to the operator's own Discord client and self-ages, so an in-place edit reads as visibly
fresh.

Output: `render_embed` gains a `location` kwarg + the two description lines; unit tests
prove the line shape, argless suppression, the `<t:>` token, and retained native timestamp.
</objective>
