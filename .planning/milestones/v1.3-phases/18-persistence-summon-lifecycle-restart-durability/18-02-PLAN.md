<objective>
Build the idempotent `!panel` summon (PANEL-01): a new operator-gated branch in
`on_message` that resolves the configured panel channel (D-04), eagerly preflights
the exact D-10 permission set (incl. `pin_messages`, NOT `manage_messages`),
scans `channel.pins()` for bot-owned panels via `_is_owned_panel` (D-03/D-05),
reuses the first match in place + deletes extras (D-06) or posts+pins a fresh
panel, and wraps every write in a `discord.Forbidden` backstop (D-09). It is a
lifecycle WRITE command that does NOT route through `dispatch_spec` (D-07).

Purpose: Give the operator a single command that always reconciles to exactly one
working pinned panel, fails loud (never silently/partially) on a permission or
channel problem, and never crashes the bot thread.
Output: the `!panel` branch in `bot.py` + its gateway-free unit coverage.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Discord user -> on_message | operator command text crosses into the write/lifecycle path |
| bot -> Discord channel | the !panel branch performs the only panel post/pin/edit/delete writes |
| preflight -> write (TOCTOU) | a permission may be revoked between the eager check and a write |

## STRIDE Threat Register (ASVS L1; block on high)

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-05 | Elevation of Privilege | `!panel` command | mitigate | `!panel` reuses the `author.id != operator_id` drop in the on_message ladder (D-07); a non-operator never reaches the branch and sees no echo (Task 1). |
| T-18-06 | Tampering | stray cleanup (`delete`) | mitigate | Marker-strict identity: only `author == bot.user` AND a `wb:` custom_id is touched/deleted (`_is_owned_panel`), never author-alone — cannot delete an unrelated bot pin (Task 2, D-05). |
| T-18-07 | Denial of Service / partial-state | preflight -> write | mitigate | Eager `permissions_for` preflight refuses BEFORE any post (no orphan, SC#4); per-write `discord.Forbidden` catch closes the TOCTOU gap; the whole branch rides the non-propagating on_message envelope so nothing reaches the scheduler (Task 1, D-09). |
| T-18-08 | Information Disclosure | logs + operator copy | mitigate | Operator-feedback copy names only permission names / a non-secret stray count / the config key; CRITICAL logs carry `channel_id` (non-secret) but never the token/appid; copy is identity-free (Task 1/2, UI-SPEC invariants). |
| T-18-09 | Denial of Service | bad/unset `panel_channel_id` | mitigate | `guild.get_channel(...) is None` -> clear log + operator message + abort, never `.pins()` on None; the bot thread survives (Task 1, D-04). |
| T-18-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs in this phase (RESEARCH Package Legitimacy Audit: N/A — no new deps). |
