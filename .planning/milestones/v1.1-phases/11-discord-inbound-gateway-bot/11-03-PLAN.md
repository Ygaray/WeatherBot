## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Discord gateway â†’ on_message | untrusted user-typed message content crosses here |
| outbound webhook â†’ channel | the bot's OWN briefing posts re-enter the same channel (feedback risk) |
| bot loop â†” executor threads | the shared TTL cache is touched across threads |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-05 | Denial of Service | feedback loop: bot replies to webhook briefing / its own reply | mitigate | `if message.author.bot: return` FIRST guard (D-04, covers webhook + self) + `!` prefix + parse_weather_command word-boundary guard â€” three backstops; CMD-07 webhook test asserts no command fires |
| T-11-06 | Denial of Service | any server member drives the OpenWeather quota | mitigate | Operator-only allowlist (`author.id != operator_id` silent return, D-05) + per-location TTLCache (CMD-06) bounds caller set to one user and key set to configured locations |
| T-11-07 | Tampering | injection via the location string | mitigate | parse_weather_command is parse-don't-validate (only strip/casefold/slice, no format/eval/shell); only configured names resolve, else UnknownLocationError |
| T-11-08 | Denial of Service (of core value) | unhandled handler exception / bad fetch crashes the process | mitigate | Whole on_message body in non-propagating try/except (D-11); error logged + generic reply, never a stack trace to the user (no token/URL in logs) |
| T-11-09 | Information disclosure | over-broad intents read all messages | mitigate | Minimal intents (guilds, guild_messages, message_content only); single private server, operator-only action |
| T-11-10 | Information disclosure | token/forecast internals leaked in logs | mitigate | structlog outcome-only fields; never log the token; generic user-facing error reply |
</threat_model>
