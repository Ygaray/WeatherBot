<objective>
Run the ONE-TIME branch-coverage audit (D-05..D-09 / SC4) over the six move-path packages, fill every
uncovered move-path branch with a characterization test, and record a clean re-run in a phase audit
log. This is the gate that proves no untaken `except`/`else` on a move path is invisible to the
observable-output goldens before any extraction begins. It runs LAST because the audit must measure the
suite WITH all Wave-1 goldens present.

Purpose: Extraction risk lives on the UNTAKEN branch side (an `except`/`else` that behaves differently
in the new package) — exactly what output-only goldens cannot see. Branch mode + a one-time fill closes
that gap. Per D-08 this is an audit recorded in the phase log, NOT a standing `fail_under` gate.

Output: `21-COVERAGE-AUDIT.md` (recorded result) + `tests/test_golden_coverage_fill.py` (the fill tests,
if any were needed).
