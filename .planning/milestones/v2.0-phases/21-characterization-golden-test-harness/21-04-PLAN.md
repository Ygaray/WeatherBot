<objective>
Pin the exception identity of every move-path error type with the D-13 two-assert pattern
(`is`-identity through the caller's import path + a frozen `(__module__, __qualname__)` tuple), so a
later re-home that changes a fully-qualified name fails loud rather than silently broadening an
`except`. This is SC3 of the phase and the structural guard for the Phase-26 registry/dispatch move.

Purpose: A broadened `except` or a renamed/re-homed error type is invisible to output goldens but
breaks the catch contract the moved modules rely on. `is`-identity + frozen tuple is the tightest
pin; `isinstance` is explicitly avoided because it permits the very broadening this guards against.

Output: `tests/test_exception_identity.py` (pure type introspection ŌĆö no snapshot files).
</objective>
