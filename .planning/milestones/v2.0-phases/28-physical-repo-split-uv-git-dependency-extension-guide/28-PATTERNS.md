### startup-version-log line in `weatherbot/cli.py` (MODIFIED, insertion) ÔÇö utility, request-response

**Analog (insertion-point + logging idiom):** `weatherbot/scheduler/wiring.py:304` ÔÇö the existing
READY-gate "online" log line, fired by `_on_online` exactly once when the startup self-check passes:
```python
daemon._log.info("weatherbot online", jobs=len(scheduler.get_jobs()))
```
This is the natural place to ALSO announce the deployed module sha (it is the once-per-boot READY
moment). The `cli.py` `main`/`run` path (lines 936-951) is the alternative insertion point ÔÇö it is
where the daemon is launched and `_configure_logging` (lines 676-695) has already run.

**Logging idiom to copy** (structlog kwargs, outcome-only, never secrets ÔÇö the house style across
`cli.py` lines 171-175, 461, 516):
```python
_log = structlog.get_logger(__name__)
...
_log.info("event name", key=value, key2=value2)
```

**Core pattern to ADD** (from RESEARCH Pattern 4 ÔÇö read PEP 610 `direct_url.json` via
`importlib.metadata`; survives a clean-venv git install, reports the *deployed* sha, and the
`editable` flag is a free dev-tree tripwire):
```python
import json
from importlib.metadata import Distribution, version

def _module_provenance() -> dict[str, str]:
    dist = Distribution.from_name("yahir-reusable-bot")
    raw = dist.read_text("direct_url.json")          # PEP 610; present for git/path installs
    info = json.loads(raw) if raw else {}
    vcs = info.get("vcs_info", {})
    return {
        "module_version": version("yahir-reusable-bot"),
        "module_sha": vcs.get("commit_id", ""),            # exact resolved sha (deploy)
        "module_ref": vcs.get("requested_revision", ""),   # the tag, e.g. v0.1.0
        "editable": str(info.get("dir_info", {}).get("editable", False)),  # True => DEV tree
    }

_log.info("module provenance", **_module_provenance())
```
**Pre-wire spike (Open Question 1):** install the module from a throwaway tag into a scratch venv and
`cat` `direct_url.json` to confirm `vcs_info.commit_id` field names on uv 0.11.19 BEFORE wiring this.

## Shared Patterns

### Logging (structlog, outcome-only, never secrets)
**Source:** `weatherbot/cli.py` `_log = structlog.get_logger(__name__)` (line 62) + every
`_log.info(...)`/`_log.error(...)` call (lines 171, 245, 461, 516); `wiring.py:304`.
**Apply to:** the startup-version-log line. Use structlog kwargs; never log the webhook URL/`appid`
(T-04-01). The provenance line logs only version/sha/ref/editable ÔÇö no secret.

### Self-proof discipline (a guard is only trustworthy if a deliberately-constructed violation trips it)
**Source:** `tests/test_import_hygiene.py` docstring (lines 12-26) + every `test_selfproof_*` pair.
**Apply to:** the new `_module_provenance()` unit test (feed it both the git-install and
editable-install `direct_url.json` shapes and assert the distinguishing behavior) and the
re-scoped import-hygiene gates (keep the synthetic-dict + litmus self-proofs that DON'T need a real
`weatherbot` package; drop/retarget the two real-import self-proofs with a documented reason).

### Hatchling wheel-packages discipline (verify the built wheel, don't trust the edit)
**Source:** `pyproject.toml` line 30 inline comment ("without this hatchling silently drops the 2nd
package") + RESEARCH Pitfall 7.
**Apply to:** BOTH pyproject edits. After each, `uv build && python -m zipfile -l dist/*.whl` and
confirm the app wheel contains ONLY `weatherbot/` and the module wheel ONLY `yahir_reusable_bot/`.

### Byte-identical-or-fail oracle (re-run the Phase-21 goldens after the split)
**Source:** the Phase-21 golden suite + the custom_id freeze test (referenced in CONTEXT canonical_refs).
**Apply to:** the Gate-1 acceptance ÔÇö re-run the full suite + goldens from the consuming app against
the pinned module; any non-empty `.ambr` diff is investigated, never rubber-stamped. Known noise:
the suite prints "2 snapshots failed" but exits 0 (syrupy quirk ÔÇö trust the exit code).
