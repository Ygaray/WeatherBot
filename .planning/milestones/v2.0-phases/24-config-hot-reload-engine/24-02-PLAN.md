## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator-edited config.toml → run_daemon → reload_engine.service_pending | The operator's local config edit is the only untrusted input; it is validated by the injected `validate_config_and_templates` (pydantic `extra="forbid"` + field validators + template allow-list) before any swap |
| SIGHUP signal handler / watchfiles observer thread → main poll loop | Off-thread reload triggers cross into the daemon; only the flag is set off-thread, reload runs on the main thread |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-24-05 | Denial of Service | run_daemon main poll loop | mitigate | The poll loop wraps `service_pending` in `try/except Exception: _log.exception(...)` so a bad config edit + SIGHUP can never crash the always-on daemon (CFG-04 keep-old end-to-end). Pinned by `test_invalid_save_keeps_old_config`. (D-08 / SC-3) |
| T-24-06 | Information Disclosure | _make_watch_filter / .env edits | mitigate | The watch filter (app-side, unchanged) rejects `.env` (zero reloads); the holder never receives secrets. A secret-file edit cannot trigger a reload that surfaces secrets. Pinned by `test_env_save_never_reloads`. (V6 / Pitfall 12) |
| T-24-07 | Tampering | excluded_ids wiring | mitigate | `__heartbeat__`/`__uvmonitor__` are supplied as an app-side `excluded_ids` frozenset the engine subtracts before diffing, so a reload never tears down the liveness/monitor jobs (a `-2` reconcile break is a tamper-equivalent loss of monitoring). Pinned by the reconcile-diff golden. (Pitfall 2) |
| T-24-08 | Tampering | job-id collision via config | accept | The app-side `_no_pipe_in_identity` validator (models.py) stays inside the concrete `Config`, unaffected by the extraction — a `|` in a location name still raises at validation. No new tampering surface. |
| T-24-SC | Tampering | npm/pip/cargo installs | accept | No package installs this phase — pure in-repo relocation; RESEARCH.md Package Legitimacy Gate = N/A. |
