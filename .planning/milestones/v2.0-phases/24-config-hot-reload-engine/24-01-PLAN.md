## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator-edited config file → ReloadEngine.reload(path) | The only untrusted input: a local operator's `config.toml` edit, read via the INJECTED concrete validator (the module never parses/validates it itself) |
| local SIGHUP / watchfiles observer thread → main poll thread | A reload trigger crosses from a signal handler / observer thread to the main thread; the module must only flag-set off-thread, never run reload work re-entrantly (D-05) |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-24-01 | Denial of Service | ReloadEngine.reload (PHASE 1 validate) | mitigate | A validator raise leaves holder+jobs untouched and re-raises (keep-old); the app's outer poll-loop `except` (Plan 02) swallows so a malformed/malicious edit + SIGHUP never crashes the always-on daemon. Pinned by the reconcile-rollback test (Task 2) and `test_invalid_save_keeps_old_config` (Plan 02). (D-08 / SC-3) |
| T-24-02 | Tampering | ReloadEngine.reload (PHASE 2 reconcile) | mitigate | All-or-nothing: a reconcile throw rolls holder + jobs back to old_cfg (restore) and re-raises — a half-applied job set can never persist. Pinned by the reconcile-throw rollback test (Task 2). (D-08) |
| T-24-03 | Elevation of Privilege (re-entrancy) | request_reload / service_pending / start_watching | mitigate | Off-thread triggers (signal handler, observer thread) only `Event.set()`; reload work runs solely on the main poll thread via service_pending(). The engine seizes no process-global SIGHUP handler. Pinned by the flag-pair servicing test (Task 2). (D-05) |
| T-24-04 | Information Disclosure | ConfigHolder[T] | accept | The holder carries the app config object ONLY and never sees secrets/.env (secrets stay behind the restart boundary); no new disclosure surface is introduced by the generalization. | T-24-SC | Tampering | npm/pip/cargo installs | accept | No package installs this phase — pure in-repo relocation; RESEARCH.md Package Legitimacy Gate = N/A (no new deps). |
