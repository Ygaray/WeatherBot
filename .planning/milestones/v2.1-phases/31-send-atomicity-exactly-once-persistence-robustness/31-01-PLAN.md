<objective>
Harden the SQLite store as the ROOT de-risker of the F01 duplicate-briefing critical: open the DB in WAL with a per-connection busy_timeout, split schema bootstrap out of the read path so "read-only" reads take no write lock (F10), and confirm the multi-step weather_onecall write is atomic. All 14 connect sites in store.py currently open with the default rollback journal, busy_timeout=0, and run executescript(_SCHEMA) — whose trailing INSERT OR IGNORE seed rows take a write lock, so a status read during a daemon write can raise `database is locked`.

Purpose: WAL + non-writing reads remove the post-delivery lock contention that makes F01 reachable — this plan ships FIRST so Plan 02's atomicity fix lands on a de-risked store.
Output: A shared `_connect()` helper, a repurposed `init_db()` that owns WAL + schema + seed rows, four read fns opened read-only, all write fns routed through `_connect()`, and store regression tests.
</objective>
